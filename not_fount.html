Status API

To use the server status API, the target server must have enable-query set to true in it's server.properties file.

All requests start with one of the following URLs, and the optional queries can be used to customize/filter the response.

URL Description
https://dags.me​/status​/<server_address> Query the status of the server at the given address, on the default port (25565)
https://dags.me​/status​/<server_address>​/<port> Query the status of the server at the given address, on the given port
Query Type Description
pretty= true/false Specify whether the returned json should be pretty printed (default is false)
uuid= true/false Specify whether UUIDs should be included in the players section of the response (default is false)
keys=

key1,key2,..

 Specify which fields should be included in the response by providing a comma-separated list of keys (default is all)

Sample URLs

  • https://dags.me/status/example.com?pretty=true
  • https://dags.me/status/example.com?pretty=true&uuid=true
  • https://dags.me/status/example.com/25575?pretty=true&keys=players,online,max,motd

    • Head API

    The head API can be used to fetch the head & helmet texture of a given player using their UUID

  • https://dags.me/head/<uuid>

    • Note

    • If no match can be made for the given UUID, the default 'Steve' texture will be returned

    • Textures are cached, so may take a few hours to update
